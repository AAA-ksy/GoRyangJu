# 1. HTML이 뭐야?
## HTML=Hyper Text Markup Language
- Hyper Text = 웹 페이지의 특정 부분과 연결할 수 있는 기능을 가진 텍스트 = 링크
- Markup Language = 정보를 구조적, 계층적으로 표현 가능한 프로그래밍 언어

## 기본구조
`<!DOCTYPE html>`  
`<html lang="ko">`  → html 문서 작성 언어 명시  
`    <head>`  → 문서의 기본 설정, 외부 스타일 시트 파일 및 js 파일 연결 등  
`        <meta charset="UTF-8">`  → 문자 인코딩 방식 설정  
`        <title>HTML</title>`  → 탭 제목  
`    </head>`  
`    <body>`  
`        <h1>Hello, HTML</h1>`  
`    </body>`  
`</html>`  
- `<!DOCTYPE html>`  
→ 문서 타입 정의(DTD)  
→ 문서가 어떤 버전으로 작성되었는지 브라우저에게 선언  
→ 문서 내 최상단에 위치

# 2. 그래서 어떻게 사용하는건데?
## <>(Tag)
- HTML에서 가장 중요하고 기본이 되는 규칙
- 무언가를 표시하기 위한 꼬리표, 이름표
- 브라우저는 태그를 인식해 페이지 출력
- 시작 태그 : <태그 이름>
- 종료 태그 : <태그 이름/>

### 태그 중첩
- 중첩되는 태그는 부모 태그 안에서 종료되어야 함
올바른 예) `<h1>Hello, <i>HTML</i></h1>`  
잘못된 예) `<h1>Hello, <i>HTML</h1></i>`

### 빈 태그(Empty Tag)
- 시작 태그만 존재하고 종료 태그가 없는 태그
ex) `<br> <img src=""> <input type="">`
- 속성을 통해 화면에 나타내거나 다른 용도로 사용되는 태그

### 제목 태그(Heading Tag)
- `<h1>~<h6>`
- 숫자가 올라갈수록 낮은 수준의 제목
- 자동 개행

### 단락 태그(Paragraph Tag)
- `<p></p>`
- 화면에는 별다른 변화가 없지만 사용 시, 코드상 잘 짜인 구조
- 자동 개행

### 줄바꿈 태그
- `<br>`
- 닫기 태그와 내용이 없는 빈 태그

### 텍스트 관련 태그들
- `<b>` : 굵게 표시
- `<i>` : 이탤릭체(기울임) 표시  
→ HTML5 버전 : 특정 이유(기술적인 용어, 외국인 문구 등)로 다른 글자와 구분하기 위해 사용
- `<u>` : 밑줄 표시
- `<s>` : 중간선 표시

### 앵커(Anchor Tag)
- 앵커, 링크, a태그 등 다양한 이름으로 불림
- 다른 문서로 이동할 수 있는 링크 생성
- href 속성 : 링크의 목적지가 되는 URL  
→ #(요소 id값) : 문서 내부의 요소로 이동  
→ 사용법 : `<a href="#some-element-id">회사 소개로 이동하기</a>`
- target 속성 : 링크된 리소스를 어디에 표시할 지 설정  
→ `_self` : 현재 화면에 표시. 기본 설정  
→ `_blank` : 새로운 창에 표시.  
→ `_parent`, `_top` : 프레임이라는 특정 조건에서만 동작(최근에 프레임을 잘 쓰지 않으므로 잘 사용 X)  
- 사용법 : `<a href="http://naver.com" taget="_blank">네이버</a>`

### 의미없이 요소를 묶기위한 태그(Container)
- 별다른 의미없이 요소를 묶기위한 태그
- `<div>` : block-level  
→ 기본적으로 한 줄을 생성하여 내용을 표현  
→ Block-Level : 부모 요소의 가로 영역에 맞게 꽉 채워져 표현되는 요소(줄바꿈)  
- `<span>` : inline-level  
→ 블록 레벨 요소의 한 줄 안에서 표현되는 요소  
→ Inline-Level : 하나의 라인 안에서 자신의 내용만큼의 박스를 만드는 요소(줄바꿈X)  

### 리스트
- `<ul>` : 순서가 없는 리스트(unordered list)  
→ `<ul`> 선언 후, `<li>`을 사용해 각 항목 표현
- `<ol>` : 순서가 있는 리스트(ordered list)  
→ `<ol`> 선언 후, `<li>`을 사용해 각 항목 표현
- `<dl>` : 용어를 설명하는 리스트(description list)  
→ `<dt>` : 용어  
→ `<dd>` : 용어 설명

### 이미지(`<img>`)
- src 속성 : 이미지 경로  
→ 절대경로 : 이미지의 전체 경로  
→ 상대경로 : 현재 위치 기준으로의 이미지 경로(./ : 현재폴더, ../ : 상위폴더)
- alt 속성 : 이미지의 대체 텍스트
- width / height 속성 : 이미지 크기 설정(대체로 CSS로 설정)  
→ 하나만 설정 시, 이미지 비율을 고려하여 자동 설정
- 이미지 종류 : GIF, JPG, PNG  
→ GIF :256색. 제한적이지만 용량이 작고, 애니멘이션과 투명 이미지 가능  
→ 높은 압축률과 자연스러운 색상 표현 가능. 사진이나 일반적인 그림에 사용. 투명 지원하지 않음  
→ jpg와 비교 시, 이미지 손실이 없고 투명과 반투명 모두 지원

### 표(Table)
- `<table>` : 표. 하나 이상의 `<tr>`로 구성
- `<tr>` : 행. 자식 태그로 `<th>, <td>` 가짐
- `<td>`, `<th>` : 데이터 셀
- `<caption>` : 표의 제목
- `<thead>` : 제목 행을 그룹화
- `<tfood>` : 바닥행을 그룹화
- `<tbody>` : 본문행을 그룹화
- `<colspan>` : 셀을 가로방향으로 병합
- `<rowspan>` : 셀을 세로방향으로 병합
- 예시 : 1행 4열짜리 표를 만든다면?  
`<table>`  
`<tr>`  
`   <td></td><td></td><td></td><td></td>`  
`</tr>`
`</table>`

## 요소(Element)
- 내용을 포함한 태그 전체

### 폼 관련 요소
: 서버에 데이터를 전달하기 위한 요소
- Input 요소 : 다양한 type 속성으로 여러 종류의 입력 양식 표현  
→ type="text" : 텍스트 입력  
→ type="password" : 공개할 수 없는 내용 입력으로 값 노출 X  
→ type="radio" : 하나의 항목만 선택할 수 있는 라디오 버튼(name 속성을 일치시켜줘야 상호배타적 선택가능)  
→ type="checkbox" : 중복 선택 가능  
→ type="file" : 파일 업로드
→ type="submit" : form값을 전송(value 속성으로 값 변경 가능)  
→ type="reset" : form값을 초기화(value 속성으로 값 변경 가능)    
→ type="button" : 개발자가 직접 커스텀 가능한 아무 기능없는 버튼(value 속성으로 값 변경 가능)    
→ type="image" : submit 기능을 가진 이미지 버튼(src, alt 속성 필요)
- Select 요소 : 선택 목록상자(콤보박스)
→ `<select>` 태그 사용  
→ `<option>` : `<select>` 태그의 하위 태그로, 항목 표시  
- Textarea 요소 : 여러 줄 텍스트 입력 상자
→ cols : 가로 크기 조절(한 줄에 들어가는 글자수)  
→ rows : 세로 크기 조절(화면에 보여지는 줄 수)  
- Button 요소 : Input 요소와 동일(submit, reset, button 타입)  
→ Button 요소는 태그 사이에 내용을 작성할 수 있으므로 자유로운 스타일 표현 가능  
- Label 요소 : form 요소의 이름과 form 요소를 명시적으로 연결
→ form 요소의 id 속성값과 `<lable>`의 for 속성값 일치
- Fieldset 요소(`<fieldset>`) : 여러 개의 폼 요소를 그룹화하여 구조적으로 만들기위해 사용
- Legend 요소(<`legend>`) : 폼 요소의 제목으로 `<fieldset>` 내부에 작성  
→ `<fieldset>`의 자식 태그로 반드시 최상단에 위치
- Form 요소 : 폼 데이터를 그룹화하여 서버에 전송  
→ action 속성 : 폼 데이터를 처리하기 위한 서버의 주소  
→ method 속성 : 데이터를 전송하는 방식 지정(get, post)

## 속성(Attribute)
- 태그 이름 뒤 이름='값'으로 구성  
ex)`<h1 id='title'>Hello, HTML</h1>`
- 태그에 추가로 정보를 제공하거나 태그의 동작이나 표현을 제어할 수 있는 설정값
- 속성의 선언 순서는 중요하지 않음
- 종류  
→ 글로벌 속성 : 모든 태그에 사용 가능  
→ 특정 태그에서만 사용 가능

## 공백(Space)
- 기본적으로 두 칸 이상의 공백 무시

## 주석(Comments Tags)
- 화면에 노출되지 않는 메모 목적으로 사용
- 사용법 : `<!-- 내용-->`

# 4. 그래서 CSS는 어떻게 사용하는데?
## CSS 문법
`h1 { color: yellow; font-size:2em; }`
- 선택자 : h1
- 속성 : color
- 값 : yellow
- 선언(속성과 값) : color: yellow, font-size: 2em
- 선언부(중괄호 포함 부분) : { color: yellow; font-size:2em; }
- 규칙(선택자+선언부) : `h1 { color: yellow; font-size:2em; }`

## CSS 적용방식
### inline 방식
- 해당 요소에 직접 스타일 속성을 사용하여 규칙을 선언하는 방법
- `<div style="color:red;"> 내용 </div>`

### internal 방식
- `<style>` 태그를 활용한 방법으로, `<head>` 태그 안에 `<style>` 태그 사용
- `<style> div {color: red;} </style>`

### External 방식
- 외부 스타일 시트 파일을 이용한 방법
- 확장자는 css
- `<head>`태그 안에 `<link>`의 href 속성을 활용하여 사용
- rel 속성 : css 파일과의 관계 명시적으로
- `<link rel="stylesheet" href="css/style.css">`

### Import 방식
- 스타일 시트 내에 다른 스타일 시트 파일을 불러오는 방식
- 성능이 좋지 않아 거의 사용 X


## 선택자(selector)
: 어떤 요소를 꾸밀건지 요소를 선택하는 항목

### 요소 선택자(태그)
- 선택자 부분에 태그 이름이 들어감
- 문서 내에 선택자에 해당하는 모든 요소에 스타일 규칙이 적용
- 쉼표를 이용해 그룹화 가능

### 전체 선택자
- 문서 내에 있는 모든 요소 선택
- `*` 사용

### Class 선택자
- 요소에 구애받지 않고 스타일 규칙을 적용할 수 있는 가장 일반적인 방법
- HTML을 수정하여 class 속성을 추가하는 방식으로 적용
- CSS : `.foo { font-size: 30px; }`(`<style>` 태그 안에 사용)
- HTML : `<p class="foo"> ... </p>`

### 다중 Class 선택자
- 공백으로 구분하여 여러 개의 class 값 적용하여 다중 Class 사용

### ID 선택자
- Class 선택자와 유사하게 id 속성을 사용하여 스타일 적용
- id는 문서 내에 유일해야 함
- CSS : `#bar { background-color: yellow; }`
- HTML : `<p id="bar"> ... </p>`

### 선택자의 조합
- 요소와 클래스의 조합 : `p.bar { ... }`
- 다중 클래스 : `.foo.bar { ... }`
- id와 class의 조합 : `#foo.bar { ... }`

### 속성 선택자
- 단순 속성으로 선택  
→ `p[class] { color: silver; }`  
→ `p[class][id] { text-decoration: underline; }`
- 정확한 속성값으로 선택  
→ `p[class="foo"] { color: silver; }`  
→ `p[id="title"] { text-decoration: underline; }`
- 부분 속성값으로 선택  
→ `[class~="bar"]` : class 속성의 값이 공백으로 구분한 "bar" 단어가 포함되는 요소 선택  
→ `[class^="bar"]` : class 속성의 값이 "bar"로 시작하는 요소 선택  
→ `[class$="bar"]` : class 속성의 값이 "bar"로 끝나는 요소 선택  
→ `[class*="bar"]` : class 속성의 값이 "bar" 문자가 포함되는 요소 선택  

### 문서 구조 선택자
- 자손 선택자 : `div span { color: red; }`
→ 선택자 사이에 아무 기호없이 공백으로 구분  
→ `<div>`의 자식 요소인 `<span>` 선택
- 자식 선택자 : `div > h1 { color: red; }`
→ 선택자 사이에 `>` 기호 사용  
- 인접 형제 선택자 : `div + p { color: red; }`
→ 선택자 사이에 `+` 기호 사용
→ 형제 관계이면서 바로 뒤에 인접해있는 요소 선택  

### 가상 선택자
: 미리 정의해놓은 상황에 적용이 되도록 약속되어있는 보이지 않는 클래스
- 클래스 이름 앞에 세미콜론(;)을 사용
- :first-child : 첫 번째 자식 요소 선택(`li:first-child { color: red; }`)
- :last-child : 마지막 자식 요소 선택(`li:last-child { color: blue; }`)
- :link : 하이퍼링크이면서 아직 방문하지 않은 앵커(`a:link { color: blue; }`)
- :visited : 이미 방문한 하이퍼링크(`a:visited { color: gray; }`)
- :focus : 현재 입력 초점을 가진 요소에 적용(`a:focus { background-color: yellow; }`)
- :hover: 마우스 포인터가 있는 요소에 적용(`a:hover { font-weight: bold; }`)
- :active : 사용자 입력으로 활성화된 요소에 적용(`a:active { color: red; }`)

## 가상 요소
:미리 정의해놓은 위치에 삽입이 되도록 약속되어있는 보이지 않는 요소
- :before : 가장 앞에 요소를 삽입
- :after : 가장 뒤에 요소를 삽입
- :first-line : 요소의 첫 번째 줄에 있는 텍스트
- :first-letter : 블록 레벨의 첫번째 문자

## 구체성
: 선택자를 얼마나 명시적으로 선언했느냐를 수치화한 것
- 구체성이 클수록 우선 적용
- 0, 1, 0, 0 : 선택자에 있는 모든 id 속성값
- 0, 0, 1, 0 : 선택자에 있는 모든 class 속상값, 기타 속성, 가상 클래스
- 0, 0, 0, 1 : 선택자에 있는 모든 요소, 가상 요소
- 0, 0, 0, 0 : 전체 선택자
- 1, 0, 0, 0 : 인라인 스타일의 구체성 값
- `!important` : 별도의 구체성은 없지만 모든 구체성을 무시하고 우선권 가짐
→ `h1 { ... }  /* 0,0,0,1 */`  
→ `body h1 { ... }  /* 0,0,0,2 */`  
→ `.grape { ... }  /* 0,0,1,0 */`  
→ `*.bright { ... }  /* 0,0,1,0 */`  
→ `p.bright em.dark { ... }  /* 0,0,2,2 */`  
→ `#page { ... }  /* 0,1,0,0 */`  
→ `div#page { ... }  /* 0,1,0,1 */`  

## 상속
- 박스 모델 관련 속성은 상속되지 않음
- 상속된 값은 아무런 구체성을 가지지 않음(0,0,0,0도 아님)

## Cascading
- 중요도 & 출처
→ 기본적으로 `!important`로 선언된 모든 규칙은 그렇지 않은 규칙보다 우선  
→ 출처는 제작자, 사용자, 사용자 에이전트로 구분  
- 우선순위 : 1. 사용자 `!important` 스타일 2. 제작자 `!important` 스타일 3. 제작자 스타일 4. 사용자 스타일 5. 사용자 에이전트 스타일
- 구체성
- 선언 순서 : 나중에 우선된 게 우선

## Contents Model
: HTML5에서는 요소들이 가지고 있는 성질에 따라 요소의 종류를 정의하는 규칙. 이 규칙들을 그룹화한 것이 콘텐츠 모델
### Metadata
: 콘텐츠의 스타일, 동작을 설정하거나 다른 문서와의 관계 등 정보를 포함하는 요소
- `<head>` 내에 들어가는 요소들 

### Flow
: 문서의 자연스러운 흐름에 의해 배치되는 요소들 포함

### Sectioning
: 문서의 구조와 관련된 요소 포함
- HTML5에 새로 생긴 태그 포함(문서의 구조, 아웃라인에 영향)

### Heading
: 각 섹션의 header를 정의하는 heading 태그

### Phrasing
: 문서의 텍스트 또는 텍스트를 꾸며주는 문단 내부 레벨로 사용되는 요소

### Embedded
: 외부 콘텐츠를 표현하는 요소
- 오디오, 비디오, 이미지 등 멀티미디어 관련 요소 해당

### Interactive
: 사용자와 상호작용을 하는 요소
- 대표적으로 form 요소

## 시멘틱 마크업
: 기계(컴퓨터, 브라우저)가 잘 이해할 수 있도록 적절한 HTML 요소를 올바르게 사용한 마크업
